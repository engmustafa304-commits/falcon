# User Stories

User stories define outcomes, not page requirements.

## Visitor

- As a visitor, I want to find relevant vehicles quickly so I can decide whether to contact a dealership.
- As a visitor, I want to understand dealer credibility so I can trust the next step.
- As a visitor, I want to switch language and country so the experience matches my context.
- As a visitor, I want contact options to be clear so I do not have to hunt for next steps.

## Customer

- As a customer, I want to track my inquiries so I know what each dealership is doing.
- As a customer, I want to save vehicles so I can compare later.
- As a customer, I want my finance request status to be clear so I know what to prepare.
- As a customer, I want my profile to prefill future requests so I do not repeat myself.

## Dealer

- As a dealer, I want one operating dashboard so I can see what needs attention today.
- As a dealer, I want inventory controls so public vehicle data stays accurate.
- As a dealer, I want lead tracking so no customer request is missed.
- As a dealer, I want white-label settings so my public presence matches my brand.

## Employee

- As an employee, I want to see assigned work only so I can focus.
- As an employee, I want customer context before contact so follow-up feels professional.
- As an employee, I want simple status updates so managers know progress.
- As an employee, I want due tasks to be obvious so I do not miss follow-ups.

## Manager

- As a manager, I want team visibility so I can balance workload.
- As a manager, I want lead response metrics so I can improve operations.
- As a manager, I want approval queues so quality stays consistent.
- As a manager, I want alerts for missed work so I can intervene early.

## Platform Admin

- As a platform admin, I want tenant health visibility so I can support the platform.
- As a platform admin, I want audited support access so I can help without violating trust.
- As a platform admin, I want country configuration controls so expansion is manageable.
- As a platform admin, I want system observability so incidents can be resolved quickly.

## Story Quality Rules

- Every story must map to a role.
- Every story must have a success state.
- Every story must respect permissions.
- Every story must identify whether it is public, customer, tenant, or platform scope.
- No story should imply UI before the flow is understood.
