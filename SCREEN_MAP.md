# Screen Map

This is a conceptual screen map, not a page build plan. Screens exist only to support user journeys.

## Public Screens

- Public discovery entry.
- Search results.
- Vehicle detail.
- Dealer profile.
- Brand detail.
- Finance information.
- Authentication entry.
- Language and country selection.

## Customer Screens

- Customer dashboard.

- Notification preferences.

## Screen Rules

- Every screen must have one primary purpose.
- Every screen must declare scope: public, customer, tenant, platform.
- Every screen must declare primary role.
- Every screen must declare required permissions.
- Every screen must have empty, loading, error, and success states before implementation.
