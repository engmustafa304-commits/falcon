# Design System

Falcon uses a premium, minimal, whitespace-first design system for enterprise automotive SaaS.

## Principles

- White space first.
- Minimal interfaces.
- No visual clutter.
- Every screen has one clear purpose.
- No unnecessary animations.
- Soft shadows.
- Rounded corners.
- Professional typography.
- Luxury automotive feeling.
- Accessibility is mandatory, not optional.

## Color System

| Token | Value | Usage |
| --- | --- | --- |
| Accent / Turquoise3 | `#43BFC7` | Accent only, primary calls to action, focus states |
| Dark | `#0F172A` | Main text, high-emphasis UI |
| Background | `#FFFFFF` | Main app background |
| Section | `#F8FAFC` | Soft page sections |
| Border | `#E5E7EB` | Minimal separators |
| Success | `#10B981` | Positive states |
| Warning | `#F59E0B` | Caution states |
| Danger | `#EF4444` | Destructive or critical states |

Turquoise must be used as an accent. It should not dominate the interface.

## Token Categories

Falcon brand tokens are defined for:

- Colors.
- Typography.
- Spacing.
- Radius.
- Shadows.
- Z-index.
- Breakpoints.

Shared token exports live in `packages/design-tokens`.

## Typography

- Arabic: Alexandria.
- English: Inter.

Typography should be calm, legible, and professional, with strong hierarchy and generous line height.

## Interface Rules

- Prefer reusable primitives and patterns.
- Prefer semantic HTML and accessible states.
- Use restrained motion only when it improves clarity.
- Avoid decorative complexity.
- Design for RTL and LTR from the beginning.
- Design for desktop and mobile from the beginning.
